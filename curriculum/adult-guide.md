# Adult Guide

## Your Role

You do not need to be the coding teacher all the time. Your main job is to help the child keep momentum.

Helpful adult support:
- Open the website or project
- Read instructions aloud when needed
- Ask simple questions like "What do you want it to do?"
- Encourage testing after small changes
- Help the child describe bugs before fixing them

Try not to:
- Take over the keyboard unless truly needed
- Correct every mistake immediately
- Turn the session into a lecture

## Good Coaching Questions

- What do you think will happen?
- Which block tells it to start?
- What changed after you tested it?
- Which part is working already?
- What is one tiny thing we can try next?

## Kid-Friendly Vocabulary

- Sequence = steps in order
- Event = a start signal
- Loop = repeat
- Conditional = choice
- Variable = memory box
- Debugging = bug hunting

## When the Child Gets Stuck

Try this order:
1. Ask them to show the bug.
2. Ask what they wanted instead.
3. Change only one thing.
4. Test again.
5. Celebrate any progress, even partial.

## When to Slow Down

Slow down if the child:
- Stops wanting to click and try things
- Cannot explain what a block does
- Feels frustrated by typing
- Needs more than one session to finish each mini-project

If that happens, repeat the same concept with a new theme instead of moving on.
